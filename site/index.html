Mosquito Alert Fast Map API

Documentation

Overview

This is a very simple API that handles only GET requests and returns static json files that are pre-processed on our server nightly. We take this approach to minimize server load for requests coming from web maps and other applications that generate high traffic. The documentation for each endpoint is based on the JSON Schema vocabulary. The JSON Schema strings can be viewed raw or as formatted tables.

Each of these endpoints returns a json string that follows the same schema (below). The data is divided by year to improve map speed by allowing asynchronous load of each file. Note that the data from previous years can change (e.g. if a participant edits and old report), so it is important to reload all of this data daily.

Coverage Endpoint

 /coverage_month_data.json

This endpoint returns aggregated data on Mosquito Alert participant locations, for use in estimating sampling effort. Locations sampled 5 times per day at random intervals and are masked on participant's devices by placing the true location on a sampling grid of 0.05 degrees longitude by 0.05 degrees latitude and sending only the sampling cell identifier to the server. The location data returned by this endpoint gives number of users per sampling cell per month, with sampling cells identified by their SW corners.
